Short-Read: HMMER + PPLACER

Introduction

This pipeline is a short-read functional placement workflow for metatranscriptome (metaT) ORFs. It assigns taxonomy by placing gene fragments onto fixed, curated reference gene trees. Instead of relying on sequence similarity alone.

  • Who is this for? Researchers working with metaT short-read ORF (contig) datasets who want: phylogenetically informed taxonomic assignments

  • Where can I run it? On a remote HPC server (e.g., HiPerGator) or local system (i.e., your computer; provided the required software is installed)

  • What’s the end goal? For each gene × metaT sample, produce a .jplace file containing phylogenetic placements. Each transcript is assigned taxonomy based on its position on the reference tree, which acts as a tally of gene-specific transcript placements.

This tutorial walks through a series of shell (.sh) scripts: Part 1 (A, B, C), Part 2, Part 3. Each corresponds to a specific stage of the pipeline. Brief descriptions are provided below.

PIPELINE STAGES
Script Purpose Main outputs
1_A Build gene-specific reference package and HMM .refpkg/, .hmm
1_B Submit parallel jobs across samples SLURM jobs
1_C Detect contigs, align, and place onto gene tree .jplace
2 Aggregate placements and export taxonomy tables *.taxID.csv
3 Classify placements and generate normalized counts *.counts.csv
